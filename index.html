Dominando los Diagramas de Casos de Uso

Profesor: Sergio Montes Repiso

El "Contrato" con el Cliente

Un diagrama de casos de uso no es solo un dibujo; es la representación visual de las funcionalidades que el sistema debe ofrecer. Es el lenguaje común entre el desarrollador y el cliente.

🔍

¿Qué es?

Un diagrama de comportamiento que describe lo que hace un sistema, no cómo lo hace.

🤝

El "Contrato"

Define el alcance del proyecto y evita malentendidos sobre las funcionalidades pactadas.

graph LR User((Usuario)) -- Interactúa --> System[Sistema]

Anatomía del Diagrama

👤

Actores

Roles externos que interactúan con el sistema. Puede ser una persona, otro sistema o incluso el tiempo.

                                graph TD
                                    A((Cliente))
                                    B((Admin))
🔵

Casos de Uso

Una unidad funcional del sistema. Representa una acción o un proceso completo que aporta valor al actor.

                                graph TD
                                    UC([Realizar Pedido])
🔗

Relaciones

Conexiones entre elementos. Incluye Asociaciones, <<include>> y <<extend>>.

                                graph LR
                                    A([Login]) -. <<include>> .-> B([Verificar Pass])
